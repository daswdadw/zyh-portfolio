# GitHub 上传部署说明

这个文件夹已经是可以直接上传到 GitHub 的最终版本。

当前文件说明：

- `index.html`：网站主页
- `styles.css`：样式文件
- `script.js`：交互脚本
- `vercel.json`：Vercel 部署配置
- `.nojekyll`：避免 GitHub Pages 使用 Jekyll 处理静态文件

## 方法一：网页上传到 GitHub

### 第 1 步：注册并登录 GitHub

如果你还没有 GitHub 账号，先去 [https://github.com](https://github.com) 注册。

### 第 2 步：创建新仓库

1. 登录 GitHub
2. 点击右上角 `+`
3. 选择 `New repository`
4. 仓库名建议填写：
   `zhouyunhui-portfolio`
5. 选择 `Public`
6. 不要勾选 `Add a README file`
7. 点击 `Create repository`

### 第 3 步：上传网站文件

1. 打开你刚创建的仓库页面
2. 点击 `uploading an existing file`
3. 把这个文件夹里的所有文件一起拖进去：
   - `index.html`
   - `styles.css`
   - `script.js`
   - `vercel.json`
   - `.nojekyll`
4. 页面下方填写提交信息，例如：
   `first portfolio upload`
5. 点击 `Commit changes`

### 第 4 步：开启 GitHub Pages

1. 进入仓库页面
2. 点击 `Settings`
3. 左侧找到 `Pages`
4. 在 `Build and deployment` 里：
   - `Source` 选择 `Deploy from a branch`
   - `Branch` 选择 `main`
   - 文件夹选择 `/root`
5. 点击 `Save`

### 第 5 步：等待网站生成

通常几十秒到几分钟后，GitHub Pages 会生成你的网站地址。

地址一般类似：

`https://你的用户名.github.io/仓库名/`

例如：

`https://yourname.github.io/zhouyunhui-portfolio/`

## 修改网站后怎么更新

如果你后面修改了文件，只需要重新进入 GitHub 仓库页面，再上传新文件覆盖旧文件即可。

## 小提醒

- `index.html` 必须放在仓库根目录
- `styles.css` 和 `script.js` 也要和 `index.html` 放在同一层
- 如果你以后加入头像或项目图片，也一起上传到仓库里
